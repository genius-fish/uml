# `genius-uml` DSL Reference

This document describes the public DSL exposed by `genius-uml`'s `core`
module, plus the `render` module's runtime façade. It is the canonical
reference: every helper that appears here is a stable part of the API,
and helpers that don't appear here are package-private implementation
detail.

The DSL is **context-function-driven** and **typed**. A diagram is built
inside a `block { ... }` body that produces a `PUmlNode` AST value. Each
helper requires its structural context (e.g. `InArchimate`, `InSequence`)
to be in scope, so misplaced calls — `shape(...)` outside an
`archimateDiagram { ... }` — fail at compile time, not at render time.

## Concepts

A diagram passes through three layers:

```
DSL helpers (context functions)  →  PUmlNode AST  →  PlantUML source
                          block { uml { ... } }       Emit.emit
```

- **`PUmlCtx`** — a per-block mutable buffer that DSL helpers append to.
  You never construct or pass one around explicitly; it lives in the
  context-function scope.
- **`PUmlNode`** — the rendered AST. Five constructors:
  - `Statement(value: String)` — a literal line.
  - `Sequence(children: Vector[PUmlNode])` — flat siblings.
  - `Block(header: String, body: PUmlNode)` — `header { body }`.
  - `Diagram(kind: String, body: PUmlNode)` — `@startKIND ... @endKIND`.
  - `Comment(text: String)` — `' text`.
- **`Emit.emit(node: PUmlNode): String`** — pure, total fold; this is
  the only renderer-free way to inspect a diagram's source.
- **Markers** — sealed trait capabilities (`InUml`, `InArchimate`, ...)
  injected by the parent block and required by domain helpers. The
  compiler enforces nesting via missing-given errors.
- **`Alias`** — an opaque, alphanumeric-only PlantUML identifier.
  Auto-generated by helpers like `shape` and `actor`; rarely constructed
  by hand.

A diagram is fully *value*-shaped: pure code in, `PUmlNode` out, no
side-effects, safe to compose, log, snapshot, or reuse.

Almost every diagram needs:

```scala
import fish.genius.uml.dsl.*                  // block, uml, primitives, skin, Alias

import fish.genius.uml.render.{Emit, PUmlEngine, Renderer, PUmlViewer, PUmlError}
```

```scala
import fish.genius.uml.testkit.{Golden, PUmlAssertions}
import fish.genius.uml.testkit.PUmlAssertions.*
```

## Quick start

```scala
import fish.genius.uml.dsl.*
import fish.genius.uml.dsl.archimate.*
import fish.genius.uml.dsl.archimate.ShapeType.*
import fish.genius.uml.render.Emit

given ArchimateConfiguration = ArchimateConfiguration()

val doc = block:
  uml:
    archimateDiagram:
      val customer = shape(BusinessActor, label("Customer"))
      val store    = shape(ApplicationComponent, label("Web store"))
      relationship(RelationshipType.Serving)(store)(customer)

println(Emit.emit(doc))
```

Outputs (abridged):

```
@startuml
!define TECHN_FONT_SIZE 12
...
rectangle #BUSINESS "Customer" <<$archimate/business-actor>> as a0
rectangle #APPLICATION "Web store" <<$archimate/application-component>> as a1
a1 -- a0 : ""
@enduml
```

## Top-level entry points

### `block(body): PUmlNode`

Allocates a fresh `PUmlCtx` and runs `body`. Returns the produced AST.
This is the *only* user-facing entry point that allocates a top-level
context — every other helper that captures a sub-tree
(`childBlock`, `expressionWithBody`, `archimateDiagram`, etc.) inherits
counters from its surrounding `PUmlCtx`, which keeps auto-generated
aliases unique across nested scopes.

```scala
val doc: PUmlNode = block:
  uml:
    statement("Alice -> Bob: hi")
```

### `uml(body): Unit` *(requires `PUmlCtx`)*

Wraps `body` in `@startuml ... @enduml`. Injects `InUml` into the body so
domain DSLs (`archimateDiagram`, `sequenceDiagram`, `activityDiagram`)
become callable.

### `diagram(kind: String)(body): Unit` *(requires `PUmlCtx`)*

Same as `uml` but with a custom kind, so you can author non-UML PlantUML
diagrams (`gantt`, `mindmap`, `wbs`, `salt`, …):

```scala
block:
  diagram("gantt"):
    statement("[Design]    lasts 2 weeks")
    statement("[Develop]   lasts 6 weeks and starts at [Design]'s end")
```

## Primitives

All primitives live in `fish.genius.uml.dsl` and require `PUmlCtx`. They
are intentionally low-level — domain helpers compose them.

| Helper | Effect |
|---|---|
| `statement(value: String)` | Emit a single literal line. |
| `statements(text: String)` | Emit each non-empty line of `text` as its own statement. |
| `comment(text: String)` | Emit `' text` (PlantUML's single-line comment). |
| `expression(build: Alias => String): Alias` | Allocate a fresh alias, build a line from it, emit, return the alias. |
| `expressionWithBody(build: Alias => String)(body): Alias` | Like `expression`, but the built line becomes the header of a `Block` whose body is `body`. |
| `headerBlock(header: String)(body)` | Emit `header { body }` without allocating an alias. |
| `when(cond: Boolean)(body)` | Sugar for `if cond then body`. |

`emit(node: PUmlNode)` is the lowest-level escape hatch: it appends the
node verbatim. Prefer typed helpers when one exists.

## Aliases

`Alias` is an opaque type wrapping a sanitised, lower-case alphanumeric
identifier safe to splice into PlantUML source. Three constructors:

| Constructor | Purpose |
|---|---|
| `Alias.fresh(using PUmlCtx)` | Auto-allocate a unique alias from the context's `Counters`. |
| `Alias(raw: String)` | Sanitise an arbitrary string (strip non-alnum, lowercase, replace digits with English words). |
| `Alias.unsafe(value: String)` | Wrap a string verbatim (caller's responsibility). |

You'll mostly receive aliases by *consuming* helpers — `shape(...)`,
`actor(...)`, `participant(...)`, `boundary(...)` all return their newly
allocated alias. Use them as references in subsequent calls:

```scala
val customer = shape(BusinessActor, label("Customer"))
val store    = shape(ApplicationComponent, label("Web store"))
relationship(RelationshipType.Serving)(store)(customer)
```

`Alias.value` extracts the underlying `String`.

Auto-allocated aliases are globally unique across nested scopes within a
single `block { ... }` invocation: the `PUmlCtx` shares its counter with
every `childBlock` it spawns. You will not see two `as a0` lines in the
same diagram unless you mint them yourself with `Alias.unsafe`.

## Markers (compile-time capabilities)

Each marker is a sealed trait introduced as a `given` by its enclosing
helper and required as a `using` by the helpers that depend on it:

| Marker | Introduced by | Required by |
|---|---|---|
| `InUml` | `uml`, `diagram` | every domain entry-point |
| `InArchimate` | `archimateDiagram` | `shape`, `boundary`, `relationship`, `defineStereoType`, `legend`, `archimateSkinParam`, `archimatePreamble` |
| `InBoundary` | `boundary` body | (informational; no helpers gate on it currently) |
| `InSequence` | `sequenceDiagram` | every participant constructor, `step`, `box`, `autonumber`, `teozRenderingEngine`, all group helpers |
| `InBox` | `box` body | (informational) |
| `InGroup` | (reserved) | — |
| `InActivity` | `activityDiagram` | `title`, `start`, `action`, `stop`, `end`, `note` |

Misplaced calls produce a *missing given* error like:

```
no given instance of type fish.genius.uml.dsl.markers.InArchimate was found
```

That is the entire enforcement model — there is no runtime check.

## Skinparams and SVG sprites

### `skinParam(stereotype: String, properties: SkinParamProperty*): Unit` *(requires `InUml`)*

Emits `skinparam <stereotype> { <props> }`. With no properties, it emits
nothing.

```scala
import fish.genius.uml.dsl.SkinParamProperty.*

skinParam(
  "Activity",
  FontColor("black"),
  BorderColor("steelblue"),
  BorderThickness(2),
)
```

### `SkinParamProperty`

Typed enum covering the most common PlantUML skinparam values. Each case
knows its property name, its value rendering, and (for style enums) the
single canonical key it maps to.

| Cases | Rendered name |
|---|---|
| `FontColor`, `BorderColor`, `BackgroundColor`, `Color` | (same as case) |
| `FontSize`, `StereotypeFontSize`, `BorderThickness` | (same as case) |
| `Shadowing(true/false)` | `Shadowing` |
| `BorderDashed`, `BorderDotted`, `BorderPlain`, `BorderBold` | `BorderStyle` |
| `StereoTypeAlignmentRight`, `…Left`, `…Center` | `StereotypeAlignment` |
| `RoundCorner(int)`, `DiagonalCorner(int)` | (same as case) |
| `EntrySeparatorColor(hex)` | `EntrySeparator` |
| `SkinProp(name, value)` | `name` (escape hatch) |

`SkinProp` exists for properties not yet modelled — supply the literal
PlantUML name and value and you're done.

### `svgIcon(svg, name = None, description = None): SvgIcon` *(requires `InUml`)*

Registers a `<svg>` sprite under an alias and returns an `SvgIcon`
handle. Use the handle inside `legend(tags = ...)` and `ShapeLabel(tags
= ...)`. If `name` is omitted a fresh alias is allocated.

```scala
val api = svgIcon("<svg>...</svg>", name = Some("api"), description = Some("Has an API"))
```

Emits: `sprite $api <svg>...</svg>`.

## Archimate DSL

### Configuration

```scala
final case class ArchimateConfiguration(
  technicalFontSize: Int = 12,
  descriptionFontSize: Int = 10
)
object ArchimateConfiguration:
  given default: ArchimateConfiguration = ArchimateConfiguration()
```

A given `ArchimateConfiguration` must be in scope when you call
`archimateDiagram`. The default is provided automatically; override it
to tune label-line font sizes.

### Entry point

```scala
def archimateDiagram(
  body: (PUmlCtx, InArchimate, ArchimateConfiguration) ?=> Unit
)(using PUmlCtx, InUml, ArchimateConfiguration): Unit
```

Emits the canonical Archimate skinparam preamble (rounded corners,
diagonal corners, default text alignment, …) and runs `body` with
`InArchimate` in scope. Named `archimateDiagram` (rather than
`archimate`) to avoid colliding with the surrounding
`fish.genius.uml.dsl.archimate` package.

### Shapes

```scala
def shape(
  shapeType: ShapeType,
  label: ShapeLabel,
  color: ShapeColor = ShapeColor(),
  stereoType: Option[ShapeStereoType] = None
): Alias
```

Emits one `rectangle <fill> "<label>" <<archimate/...>> as <alias>`
line. Mass-objects (`ApplicationDataObject`, `BusinessObject`,
`BusinessContract`, `BusinessRepresentation`, `TechnologyArtifact`,
`PhysicalMaterial`) automatically get a `----\n ` prefix in their label
to render the Archimate top-edge separator.

### `ShapeLabel`

```scala
final case class ShapeLabel(
  name: String,
  componentType: Option[String] = None,
  componentTypeDetails: Option[String] = None,
  description: Option[String] = None,
  tags: List[SvgIcon] = Nil,
)(using config: ArchimateConfiguration)
```

The composite caption rendered inside a shape:

```
<name>
<size:tech>[<componentType>: <componentTypeDetails>]</size>

 <size:desc><description></size>

<icon0><icon1>…
```

Use the `label(...)` helper as a shorter alternative to constructing
the case class directly:

```scala
val l = label(
  "Order management",
  componentType = Some("IT system"),
  description   = Some("primary OMS"),
)
```

### `ShapeColor` and `color`

```scala
final case class ShapeColor(hex: Option[String] = None)
def color(hex: String): ShapeColor
```

`color("FF0000")` and `color("#FF0000")` are equivalent. Default is to
fall back to the layer's group default colour.

### `ShapeType` (the catalogue)

61 cases across 7 layers. Each declares its layer, its sprite-pack name,
and (via `decorate`) any label adjustments. All cases:

| Layer | Cases |
|---|---|
| **Application** | `ApplicationComponent`, `ApplicationService`, `ApplicationCollaboration`, `ApplicationInterface`, `ApplicationFunction`, `ApplicationInteraction`, `ApplicationProcess`, `ApplicationEvent`, `ApplicationDataObject` |
| **Strategy** | `StrategyResource`, `StrategyCapability`, `StrategyValueStream`, `StrategyCourseOfAction` |
| **Business** | `BusinessActor`, `BusinessRole`, `BusinessCollaboration`, `BusinessInterface`, `BusinessProcess`, `BusinessFunction`, `BusinessInteraction`, `BusinessEvent`, `BusinessService`, `BusinessObject`, `BusinessContract`, `BusinessRepresentation`, `BusinessProduct`, `BusinessLocation` |
| **Technology** | `TechnologyNode`, `TechnologyDevice`, `TechnologySystemSoftware`, `TechnologyCollaboration`, `TechnologyInterface`, `TechnologyPath`, `TechnologyCommunicationNetwork`, `TechnologyFunction`, `TechnologyProcess`, `TechnologyInteraction`, `TechnologyEvent`, `TechnologyService`, `TechnologyArtifact` |
| **Physical** | `PhysicalFacility`, `PhysicalEquipment`, `PhysicalDistributionNetwork`, `PhysicalMaterial` |
| **Motivation** | `MotivationStakeholder`, `MotivationDriver`, `MotivationAssessment`, `MotivationGoal`, `MotivationOutcome`, `MotivationPrinciple`, `MotivationRequirement`, `MotivationConstraint`, `MotivationMeaning`, `MotivationValue` |
| **Implementation** | `ImplementationWorkPackage`, `ImplementationDeliverable`, `ImplementationPlateau`, `ImplementationEvent`, `ImplementationGap` |

Static accessors:

- `ShapeType.application` / `.strategy` / `.business` / `.technology` /
  `.physical` / `.motivation` / `.implementation` — per-layer lists.
- `ShapeType.*` — flat list of all 61.

### `ShapeGroup` (layers)

`Application`, `Strategy`, `Business`, `Technology`, `Physical`,
`Motivation`, `Implementation`. Each declares its `defaultColor` (e.g.
`#APPLICATION`) and sprite-pack `prefix` (e.g. `application`).

### Boundaries

```scala
def boundary(
  label: String,
  boundaryType: Option[String] = None,
  shapeType: Option[ShapeType] = None,
  color: ShapeColor = ShapeColor(),
  stereoType: Option[ShapeStereoType] = None,
)(body: (PUmlCtx, InArchimate, InBoundary, ArchimateConfiguration) ?=> Unit): Alias
```

Emits a `rectangle "==<label>" <<...>> as <alias> { ... }` block.
Boundaries can nest. The body sees `InArchimate` so all archimate
helpers remain available; `InBoundary` is also injected in case you
want to gate further helpers on it.

### Stereotypes

```scala
def defineStereoType(
  name: String,
  description: Option[String],
  legendColor: Option[String],
  skinParameters: SkinParamProperty*
): ShapeStereoType
```

Registers a stereotype, emits the matching `skinparam rectangle<<NAME>>`
block, and returns a handle you pass to `shape(..., stereoType = ...)`
or `legend(stereoTypes = ...)`. The stereotype name is uppercased.

### Legends

```scala
def legend(
  title: String = "Legend",
  backgroundColorHexOrName: String = "#GhostWhite",
  stereoTypes: List[ShapeStereoType] = Nil,
  tags: List[SvgIcon] = Nil,
): Unit
```

Emits a `legend bottom right ... endlegend` block, including a header
row, one row per stereotype, and one row per SVG-icon tag.

### Relationships

```scala
def relationship(
  ofType: RelationshipType,
  edge: Edge = Edge.NoDirection,
  label: Option[String] = None,
): Alias => Alias => Unit
```

Curried so a relationship template can be applied to many `(source,
target)` pairs:

```scala
val composes = relationship(RelationshipType.Composition, Edge.Down, Some("composes"))
composes(parent)(childA)
composes(parent)(childB)
```

### `RelationshipType` (15 cases)

Each declares the PlantUML connector tokens that surround the `Edge`
direction:

| Type | Connector |
|---|---|
| `Composition` | `*-...-` |
| `Aggregation` | `o-...-` |
| `Assignment` | `@@-...->>` |
| `Serving` | `-...->` |
| `Flow` | `....>>` |
| `Specialization` | `-...-|>` |
| `Association` | `=...=` |
| `DirectedAssociation` | `=...=>` |
| `Realization` | `~...~|>` |
| `Triggering` | `-...->>` |
| `Access` | `~...~` |
| `ReadAccess` | `<-...~` |
| `WriteAccess` | `~...->` |
| `ReadWriteAccess` | `<~...~>` |
| `Influence` | `....>` |

`RelationshipType.*` is the flat list.

### `Edge` (5 directions)

`NoDirection` (empty), `Up`, `Down`, `Left`, `Right`. Renders into the
middle of a relationship's connector token (e.g. `*-DOWN-`).

### Helpers

- `archimateSkinParam(stereotype, props*)` — typed shortcut equivalent to
  `skinParam`, but reusable from inside an `archimateDiagram` body
  without re-summoning `InUml`.
- `archimatePreamble` — the full canonical preamble, exposed in case you
  want to call it again after a custom theme.

## Sequence diagram DSL

### Entry point

```scala
def sequenceDiagram(body: (PUmlCtx, InSequence) ?=> Unit)(using PUmlCtx, InUml): Unit
```

### Pragmas

| Helper | Emits |
|---|---|
| `autonumber()` | `autonumber` |
| `teozRenderingEngine()` | `!pragma teoz true` |

### Participants

All eight participant constructors share the signature
`(title: String, color: Option[String] = None): Alias`:

| Helper | Plant UML type |
|---|---|
| `actor` | `actor` |
| `boundary` | `boundary` |
| `control` | `control` |
| `entity` | `entity` |
| `database` | `database` |
| `collections` | `collections` |
| `queue` | `queue` |
| `participant` | `participant` |

Pass a colour as `Some("#1234AB")` to tint a single participant. The
underlying helper `participantOf(participantType, title, color)` is also
public if you need a custom PlantUML participant type.

### Boxes

```scala
def box[A](
  title: String,
  color: Option[String] = None
)(body: (PUmlCtx, InSequence, InBox) ?=> A): A
```

Emits `box "<title>" <color>` ... `end box` (no braces — PlantUML
rejects brace-delimited boxes). Returns the body's value, so you can
capture participant aliases declared inside:

```scala
val (browser, api) = box("Front-end", Some("#DAEEFF")):
  (participant("Browser"), participant("API"))
```

### Steps

```scala
def step(title: String)(from: Alias)(to: Alias): Unit
```

Emits `<from>-><to>: <title>`. Quotes and newlines in the title are
escaped (`"` is stripped, `\n` becomes `\\n`).

### Control structures

| Helper | Emits |
|---|---|
| `altGroup(title)` | `alt <title>` |
| `elseGroup(title)` | `else <title>` |
| `loopGroup(title)` | `loop <title>` |
| `optGroup(title)` | `opt <title>` |
| `parGroup(title)` | `par <title>` |
| `breakGroup(title)` | `break <title>` |
| `criticalGroup(title)` | `critical <title>` |
| `group(title)` | `group <title>` |
| `endGroup()` | `end` |

All eight openers take `Option[String] = None`. There is no
auto-closing — pair every opener with `endGroup()` exactly as in
PlantUML. Misnesting produces a malformed diagram, not a compile error.

### Example

```scala
sequenceDiagram:
  autonumber()
  val user = actor("User")
  val (browser, api) = box("Front-end", Some("#DAEEFF")):
    (participant("Browser"), participant("API"))

  step("Open store")(user)(browser)
  step("Click checkout")(browser)(api)
  loopGroup(Some("for each item"))
  step("Reserve inventory")(api)(browser)
  endGroup()
```

## Rendering

### `Emit.emit(node: PUmlNode): String`

Pure source-level renderer. Use this when you want to inspect, hash,
golden-test, or pipe the PlantUML source somewhere else without invoking
the actual rendering engine.

### `PUmlEngine` (ZIO 2 service)

```scala
trait PUmlEngine:
  def render(doc: PUmlNode, filename: String, format: FileFormat): ZIO[Scope, PUmlError, os.Path]
  def renderBytes(doc: PUmlNode, format: FileFormat): ZIO[Any, PUmlError, Array[Byte]]
```

Two layers:

- `PUmlEngine.live` — wraps PlantUML's `SourceStringReader.outputImage`.
  The output file lives inside a `Scope`-managed temp workspace and is
  deleted when the scope closes.
- `PUmlEngine.test` — captures the `.puml` source to a scoped temp dir
  with a placeholder for the requested format. Useful for unit tests
  that don't want to drag the renderer in.

`liveWith(config: PUmlConfig)` swaps in a custom config (the only
current setting is `workspaceRoot`).

### Accessor methods on `PUmlEngine`

Convenience wrappers for the canonical formats:

- `PUmlEngine.renderSvg(doc, filename = "diagram")`
- `.renderPng(...)`
- `.renderEps(...)`
- `.renderLatex(...)` — emits LaTeX without preamble
- `.renderLatexFull(...)` — emits LaTeX with full preamble

### `Renderer` (façade)

A typed top-level façade mirroring the original `fish.genius.uml.Renderer`
shape:

```scala
import fish.genius.uml.render.Renderer

ZIO.scoped(Renderer.svg(doc))     // also: .png, .eps, .latex, .latexFull, .render(format)
```

Plus extension methods on `PUmlNode`:

```scala
doc.renderedAsSvg()
doc.renderedAsPng()
doc.renderedAsEps()
doc.renderedAsLatex()
```

### `PUmlError`

```scala
enum PUmlError extends Throwable:
  case RenderFailed(cause: Throwable)
  case ViewerFailed(cause: Throwable)
  case Internal(cause: Throwable)
```

### `PUmlViewer.open(file: os.Path)`

Opens a rendered file with the OS's default viewer (`xdg-open` on Linux,
`open` on macOS, `explorer` on Windows). Useful for example apps and
side-by-side eyeballing during development.

### Example

```scala
val program: ZIO[PUmlEngine, PUmlError, Array[Byte]] =
  PUmlEngine.renderBytes(doc, FileFormat.SVG)

val live = program.provide(PUmlEngine.live)
```

## Testing helpers

### `Golden.assertMatches`

```scala
def assertMatches(
  id: String,
  actual: String,
  resourcesDir: os.Path,
  extension: String = "puml",
  update: Boolean = shouldUpdate,
): ZIO[Any, IOException, TestResult]
```

Compare `actual` to `<resourcesDir>/golden/<id>.<extension>`. When
`update` is true (or `ZIO_TEST_UPDATE_GOLDEN` is set in the env), the
file is created/overwritten and the assertion passes vacuously. When
the file is missing and `update` is false, the test fails with a hint
to set the env var.

`Golden.moduleResources("core")` resolves to
`<workspace>/core/test/resources` (uses `MILL_WORKSPACE_ROOT` if set,
otherwise `os.pwd`).

### `PUmlAssertions`

Two flavours, both imported from `PUmlAssertions.*`:

**Assertion combinators** (for the classic `assert(value)(assertion)`
pattern):

- `containsStatement(needle)`
- `containsBlock(headerNeedle)`
- `statementCountIs(needle, expected)`
- `hasDiagramKind(kind)`

**Extension methods on `PUmlNode`** (for `assertTrue`):

- `n.countStatement(needle: String): Int`
- `n.hasBlock(headerNeedle: String): Boolean`
- `n.statements: List[String]`
- `n.statementsContaining(needle: String): List[String]`
- `n.diagramKinds: List[String]`

Example:

```scala
import fish.genius.uml.testkit.PUmlAssertions.*

assertTrue(
  doc.countStatement("Alice") == 2,
  doc.hasBlock("rectangle"),
  doc.diagramKinds == List("uml"),
)
```

## Patterns and tips

### Capture aliases out of nested blocks

`block { ... }`, `boundary { ... }`, `box { ... }`, and
`expressionWithBody` all return values. Use them to lift aliases out of
the body:

```scala
val (browser, api) = box("Front-end"):
  (participant("Browser"), participant("API"))
```

### Reuse a relationship template

`relationship(ofType, edge, label)` is curried. Build it once, apply
many times:

```scala
val composes = relationship(RelationshipType.Composition)
composes(parent)(childA)
composes(parent)(childB)
composes(parent)(childC)
```

### Skip the renderer in tests

For round-trip tests, skip `PUmlEngine.live` and assert on the AST
directly with `PUmlAssertions`, or compare `Emit.emit(doc)` against a
checked-in golden via `Golden.assertMatches`. Both are pure and fast.

### Wrap your own helpers

The DSL is just functions. Wrap recurring snippets in your own
extensions:

```scala
def microservice(name: String, language: String)(using PUmlCtx, InArchimate, ArchimateConfiguration): Alias =
  shape(
    ApplicationComponent,
    label(name, componentType = Some(language), description = Some("microservice")),
  )
```

Calling `microservice("OrderService", "Scala 3")` is now a single line
with the same compile-time guarantees as `shape`.

### Custom skinparam blocks

When a skin needs a property `SkinParamProperty` doesn't model, use
`SkinProp(name, value)`:

```scala
import fish.genius.uml.dsl.SkinParamProperty.*

skinParam("Activity",
  SkinProp("DiamondBackgroundColor", "#FFD"),
  FontColor("black"),
)
```

### Non-UML PlantUML diagrams

`uml { ... }` is just `diagram("uml") { ... }`. For Gantt charts, mind
maps, salt mockups, etc., switch the kind:

```scala
diagram("salt"):
  statement("{")
  statement("Login    | \"\"")
  statement("Password | \"\"")
  statement("[Cancel] | [  OK  ]")
  statement("}")
```

Every primitive in `fish.genius.uml.dsl` works without `InUml`-specific
gating except the domain DSLs themselves.
