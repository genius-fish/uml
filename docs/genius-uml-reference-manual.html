Overview

genius-uml lets you author PlantUML diagrams as ordinary, type-checked Scala 3 code. A diagram is a value: pure code goes in, an immutable PUmlNode AST comes out, and that AST renders to PlantUML source — and from there to SVG, PNG, EPS, or LaTeX.

This manual is split for two audiences. Each section is tagged:

Users write diagrams with the DSL — emphasis on imports, helpers, examples, and gotchas.
Contributors extend the library — emphasis on internals, invariants, and how to add a dialect.

How a diagram flows through the system

// DSL helpers (context functions)  →  PUmlNode AST  →  PlantUML source  →  rendered file
block { uml { … } }                 Emit.emit          PUmlEngine.live

core

no ZIO · no os-lib · no PlantUML

The DSL, the PUmlNode AST, and the pure Emit renderer. Depends only on sourcecode.

render

depends on core

ZIO 2 service turning the AST into SVG/PNG/EPS/LaTeX via the PlantUML jar.

testkit

depends on core

Golden-file and structural assertions for ZIO Test — no renderer required.

examples

depends on core + render

Runnable end-to-end showcases (ArchiMate, sequence, activity).

The one architectural rule core has zero runtime dependencies. Anything touching the filesystem, ZIO, or PlantUML belongs in render. This keeps source generation (Emit.emit) cheap, pure, and usable in tests without the rendering pipeline.

Concepts Users

Four ideas carry the whole library.

PUmlCtx — the per-block buffer

A mutable buffer that DSL helpers append to. You never construct or pass one explicitly — it lives in the context-function scope and is supplied as a given. Every block { … } gets a fresh one, so diagrams are independent and concurrent renders are race-free by construction.

PUmlNode — the AST

The rendered diagram is one of five constructors:

ConstructorRenders as
Statement(value: String)a literal line
Sequence(children: Vector[PUmlNode])flat siblings
Block(header: String, body: PUmlNode)header { body }
Diagram(kind: String, body: PUmlNode)@startKIND … @endKIND
Comment(text: String)' text

Emit.emit(node): String — the pure renderer

A total fold over the AST. This is the only renderer-free way to inspect a diagram's source — ideal for hashing, golden tests, or piping PlantUML somewhere else.

Markers — compile-time capabilities

Sealed-trait capabilities (InUml, InArchimate, InSequence, InActivity, …) are injected as givens by the parent block and required as using by domain helpers. Calling shape(…) outside an archimateDiagram { … } is a compile error, not a runtime surprise.

Diagrams are values A PUmlNode has no side effects. It is safe to compose, log, snapshot, reuse, or pass around like any other immutable data.

A first diagram

given ArchimateConfiguration = ArchimateConfiguration()

val doc = block:
  uml:
    archimateDiagram:
      val customer = shape(BusinessActor, label("Customer"))
      val store    = shape(ApplicationComponent, label("Web store"))
      relationship(RelationshipType.Serving)(store)(customer)

println(Emit.emit(doc))

Produces (abridged):

@startuml
!define TECHN_FONT_SIZE 12
…
rectangle #BUSINESS "Customer" <<$archimate/business-actor>> as a0
rectangle #APPLICATION "Web store" <<$archimate/application-component>> as a1
a1 -- a0 : ""
@enduml
Run it live PLANTUML_AVAILABLE=1 ./mill examples.runMain fish.genius.uml.examples.ArchimateExample renders a real file. Without the env var the examples only print source to stdout.

Top-Level Entry Points Users

block(body): PUmlNode

Allocates a fresh PUmlCtx, runs body, and returns the produced AST. The only user-facing entry point that allocates a top-level context. Every nested capture (childBlock, archimateDiagram, box, …) inherits counters from the surrounding context, keeping auto-generated aliases unique across scopes. 

val doc: PUmlNode = block:
  uml:
    statement("Alice -> Bob: hi")

uml(body): Unit requires PUmlCtx

Wraps body in @startuml … @enduml and injects InUml, which is what makes the domain DSLs (archimateDiagram, sequenceDiagram, activityDiagram) callable.

diagram(kind: String)(body): Unit requires PUmlCtx

Same as uml but with a custom kind, for non-UML PlantUML diagrams (gantt, mindmap, wbs, salt, …).

block:
  diagram("gantt"):
    statement("[Design]  lasts 2 weeks")
    statement("[Develop] lasts 6 weeks and starts at [Design]'s end")

Primitives Users

All primitives live in fish.genius.uml.dsl and require PUmlCtx. They are intentionally low-level — the domain helpers compose them.

HelperEffect
statement(value: String)Emit a single literal line.
statements(text: String)Emit each non-empty line of text as its own statement.
comment(text: String)Emit ' text (PlantUML single-line comment).
expression(build: Alias => String): AliasAllocate a fresh alias, build a line from it, emit, return the alias.
expressionWithBody(build: Alias => String)(body): AliasLike expression, but the built line becomes a Block header over body.
headerBlock(header: String)(body)Emit header { body } without allocating an alias.
when(cond: Boolean)(body)Sugar for if cond then body.
emit(node: PUmlNode)Lowest-level escape hatch: append the node verbatim. Prefer a typed helper when one exists.

Aliases Users

Alias is an opaque type wrapping a sanitised, lower-case alphanumeric identifier safe to splice into PlantUML source. Three constructors:

ConstructorPurpose
Alias.fresh(using PUmlCtx)Auto-allocate a unique alias from the context's Counters.
Alias(raw: String)Sanitise an arbitrary string (strip non-alnum, lowercase, replace digits with English words).
Alias.unsafe(value: String)Wrap a string verbatim (caller's responsibility).

You'll mostly receive aliases from consuming helpers — shape, actor, participant, boundary all return their newly allocated alias. Use them as references in later calls; Alias.value extracts the underlying String.

Globally unique by default Auto-allocated aliases are unique across nested scopes within a single block { … }: the PUmlCtx shares its counter with every child block it spawns. You will not see two as a0 lines unless you mint them yourself with Alias.unsafe.

ArchiMate DSL Users

Configuration

  technicalFontSize: Int = 12,
  descriptionFontSize: Int = 10,

object ArchimateConfiguration:
  given default: ArchimateConfiguration = ArchimateConfiguration()

A given ArchimateConfiguration must be in scope when you call archimateDiagram. The default is provided automatically; override it to tune label-line font sizes.

Entry point

def archimateDiagram(
  body: (PUmlCtx, InArchimate, ArchimateConfiguration) ?=> Unit

Emits the canonical ArchiMate skinparam preamble and runs body with InArchimate in scope. Named archimateDiagram (not archimate) to avoid colliding with the surrounding fish.genius.uml.dsl.archimate package.

Shapes

Boxes

  title: String,
  color: Option[String] = None,
)(body: (PUmlCtx, InSequence, InBox) ?=> A): A

Emits box "<title>" <color> … end box (no braces — PlantUML rejects brace-delimited boxes). Returns the body's value, so you can capture aliases declared inside:

val (browser, api) = box("Front-end", Some("#DAEEFF")):
  (participant("Browser"), participant("API"))

Steps

def step(title: String)(from: Alias)(to: Alias): Unit

Emits <from>-><to>: <title>. Quotes and newlines in the title are escaped (" is stripped, \n becomes \\n).

Rendering Users

Emit.emit(node: PUmlNode): String

Pure source-level renderer. Use it to inspect, hash, golden-test, or pipe PlantUML source without invoking the engine. Lives in core.

PUmlEngine — the ZIO 2 service

trait PUmlEngine:
  def render(doc: PUmlNode, filename: String, format: FileFormat): ZIO[Scope, PUmlError, os.Path]
  def renderBytes(doc: PUmlNode, format: FileFormat): ZIO[Any, PUmlError, Array[Byte]]

Two layers:

  • PUmlEngine.live — wraps PlantUML's SourceStringReader.outputImage. The output file lives inside a Scope-managed temp workspace and is deleted when the scope closes.
  • PUmlEngine.test — captures the .puml source to a scoped temp dir with a placeholder for the requested format. For unit tests that don't want the renderer on the classpath.

liveWith(config: PUmlConfig) swaps in a custom config (currently just defaultFormat and workspaceRoot).

Format accessors

  • PUmlEngine.renderSvg(doc, filename = "diagram")
  • .renderPng(...), .renderEps(...)
  • .renderLatex(...) — LaTeX without preamble
  • .renderLatexFull(...) — LaTeX with full preamble

Renderer — the façade

import fish.genius.uml.render.Renderer

ZIO.scoped(Renderer.svg(doc))   // also: .png, .eps, .latex, .latexFull, .render(format)

// Plus extension methods on PUmlNode:
doc.renderedAsSvg()
doc.renderedAsPng()
doc.renderedAsEps()
doc.renderedAsLatex()

PUmlError

enum PUmlError extends Throwable:
  case RenderFailed(cause: Throwable)   // PlantUML threw while parsing/rendering
  case ViewerFailed(cause: Throwable)   // external preview command failed
  case Internal(cause: Throwable)       // filesystem / IO / JVM-side

PUmlViewer.open(file: os.Path)

Opens a rendered file with the OS default viewer (xdg-open on Linux, open on macOS, explorer on Windows). Handy for example apps and eyeballing during development.

Example

val program: ZIO[PUmlEngine, PUmlError, Array[Byte]] =
  PUmlEngine.renderBytes(doc, FileFormat.SVG)

val live = program.provide(PUmlEngine.live)

Output Layout Users Contributors

The raw engine renders into a scoped temp workspace that is deleted when the Scope closes — copy a render out if you want to keep it. DiagramOutput does exactly that, persisting to a stable, git-ignored location: 

out/<format>/<category>/<name>.<suffix>
  • <format> — file suffix without the dot: svg, png, eps, tex.
  • <category> — DiagramOutput.Test ("test") or DiagramOutput.Examples ("examples").
  • <root> — MILL_WORKSPACE_ROOT, falling back to the current working directory.
import fish.genius.uml.render.{DiagramOutput, PUmlEngine}
import net.sourceforge.plantuml.FileFormat

// Renders and persists to <root>/out/svg/examples/web-store.svg
val program = ZIO.scoped(
  DiagramOutput.save(diagram, DiagramOutput.Examples, "web-store", FileFormat.SVG)
)

├── svg/

│   └── examples/ activity-example.svg

Test suites leave their diagrams under out/<format>/test; the examples under out/<format>/examples. The whole out/ tree is git-ignored.

Testing Helpers Users

Golden.assertMatches

def assertMatches(
  id: String,
  actual: String,
  resourcesDir: os.Path,
  extension: String = "puml",
  update: Boolean = shouldUpdate,

Compares actual to <resourcesDir>/golden/<id>.<extension>. When update is true (or ZIO_TEST_UPDATE_GOLDEN is set), the file is created/overwritten and the assertion passes vacuously. When the file is missing and update is false, the test fails with a hint to set the env var.

Internals Contributors

The DSL is built from three mechanisms. Understanding them is enough to extend any part of the library.

1. Context functions carry the builder

The DSL type is PUml[A] = PUmlCtx ?=> A. block(body) allocates a fresh PUmlCtx — a mutable ArrayBuffer[PUmlNode] plus a Counters for alias allocation — supplies it as a given, runs body, then collapses the buffer into a single PUmlNode (a lone child, or a Sequence). Every helper takes the PUmlCtx implicitly and appends to its buffer via emit(node). No global state; each block is its own arena.

2. Markers enforce nesting at compile time

Marker capabilities are sealed traits in dsl/markers.scala. A wrapper (uml, archimateDiagram, …) introduces its marker as a given inside the body's context-function type; the helpers that belong to that scope demand it with using. A misplaced call fails to compile: 

no given instance of type fish.genius.uml.dsl.markers.InArchimate was found

That is the entire enforcement model — there is no runtime check.

MarkerIntroduced byRequired by
InUmluml, diagramevery domain entry-point
InArchimatearchimateDiagramshape, boundary, relationship, defineStereoType, legend, archimateSkinParam, archimatePreamble
InBoundaryboundary bodyinformational (no helpers gate on it yet)
InSequencesequenceDiagramparticipant constructors, step, box, autonumber, teozRenderingEngine, all group helpers
InBoxbox bodyinformational
InGroupreserved—
InActivityactivityDiagramtitle, start, action, stop, end, note

3. Counters keep aliases unique & deterministic

Each top-level PUmlCtx owns a Counters instance; child contexts inherit it. Alias.fresh pulls the next index, producing a deterministic ascending sequence (a0, a1, …) that is stable across reruns — which is what makes golden tests on Emit.emit output reproducible.

The render path end to end

  1. Emit.emit(doc) folds the AST into a PlantUML String (pure, in core).
  2. PUmlEngine.Live hands that string to PlantUML's SourceStringReader, writing the chosen FileFormat into a Workspace (temp dir managed by ZIO.acquireRelease).
  3. PUmlConfig supplies the default format and optional workspace root.
  4. DiagramOutput.save copies the scoped render out to out/<format>/<category> via a package-private copyOut seam, so filesystem failures surface as PUmlError.Internal.

Extending the Library Contributors

Add a new diagram dialect

To add, say, a state-machine DSL, follow the pattern the three existing dialects share:

  1. Define a marker. Add sealed trait InState to dsl/markers.scala.
  2. Write an entry point that requires InUml and introduces InState in the body's context-function type: 
    def stateDiagram(body: (PUmlCtx, InState) ?=> Unit)(using PUmlCtx, InUml): Unit =
  given InState = new InState {}
  body
  3. Write helpers that require InState and call the existing primitives (statement, expression, headerBlock, …) rather than touching the buffer directly.
  4. Return aliases from any helper that declares a node a later call will reference — reuse Alias.fresh so uniqueness is automatic.

Wrap recurring snippets — no library change needed

The DSL is just functions; users and contributors alike compose new helpers with the same compile-time guarantees:

def microservice(name: String, language: String)(using PUmlCtx, InArchimate, ArchimateConfiguration): Alias =
  shape(
    ApplicationComponent,
    label(name, componentType = Some(language), description = Some("microservice")),
  )

microservice("OrderService", "Scala 3") is now one line with shape's guarantees.

Conventions enforced in CI .scalafmt.conf (maxColumn 100; 120 for **/test/** and *Spec.scala) and .scalafix.conf (OrganizeImports, RemoveUnused, …) are enforced. Run make validate (fix + fmt + compile + test) before committing. New files usually need ./mill <module>.reformat.

Non-UML PlantUML diagrams

uml { … } is just diagram("uml") { … }. For Gantt charts, mind maps, or salt mockups, switch the kind:

diagram("salt"):
  statement("{")
  statement("Login    | \"\"")
  statement("Password | \"\"")
  statement("[Cancel] | [  OK  ]")
  statement("}")

Every primitive in fish.genius.uml.dsl works without InUml-specific gating except the domain DSLs themselves.
